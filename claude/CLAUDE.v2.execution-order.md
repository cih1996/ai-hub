§1 规则层级与优先级（统一口径）

规则按优先级从低到高分三层，高层可覆盖低层：

| 层级 | 位置 | 作用域 | 修改方式 |
|------|------|--------|----------|
| 全局规则 | {{RULES_DIR}}/CLAUDE.md + rule-*.md | 所有会话 | 通过「规则管理」Skill API |
| 团队规则 | ~/.ai-hub/teams/<group_name>/rules/*.md | 同一团队会话 | 通过「规则管理」Skill 或 Read/Edit |
| 会话规则 | 会话级 system_prompt | 仅当前会话 | 通过「规则管理」Skill API |

说明：
- 会话是执行边界，团队是共享边界。
- 团队规则写通用协作协议；会话规则写角色专属职责。

§2 固定执行顺序（必须遵循）

每次收到消息，先走以下顺序，禁止跳步：

1. 判定消息类型（分流）
- P0 会话维持：在吗、收到请回复、简短确认
- P1 执行任务：排障、部署、改配置、调度
- P2 复盘沉淀：总结、纠错、提炼经验、更新规则

2. 执行策略
- P0：直接简短回复。禁止检索知识库/记忆库，禁止调度子会话。
- P1：进入 §3「执行前读取协议」。
- P2：进入 §4「执行后写入协议」。

3. 结果回传
- 先给用户结论，再给关键证据。
- 禁止只报过程，不报结果。

---

§3 执行前读取协议（仅 P1）

3.1 上下文锁定（先于一切）
- 必须先确认：任务对象、目标范围、前置约束（如有）。
- 若本轮消息未明确但会话已有稳定上下文，则沿用并在内部锁定。
- 若出现冲突信息，先澄清，禁止猜测执行。

3.2 读取顺序（固定）
1) 会话规则（角色边界、回调对象、权限）
2) 团队级记忆/知识（当前团队共享上下文）
3) 全局记忆/知识（通用方法、历史纠错）
4) 相关 Skill 手册（若要调用 API/脚本）

3.3 Skill 前置判定（固定）
- 进入执行前，先判断当前动作是否已有对应 Skill。
- 若有对应 Skill：先按 Skill 执行，再补充会话内判断。
- 若无对应 Skill：按通用流程执行，并记录“可沉淀 Skill”的候选项。

3.4 读取触发条件
- 命令执行、配置变更、发布、跨会话调度：必须读取。
- 纯确认类回复、礼貌问候：不读取。

§4 执行后写入协议（P1 完成后 + P2）

4.1 必须写入的触发场景
- 发生错误并完成根因定位
- 发现新工具/新命令并验证可复用
- 规则被纠错（会话/团队/全局）
- 调度出现误路由并完成修复
- 上下文压缩前的关键状态快照

4.2 写入层级判定
- 团队私有事实或经验：写团队级（<group_name>/memory 或 knowledge）
- 跨团队可复用方法：写全局级（memory 或 knowledge）
- 临时过程笔记：写 notes

4.3 写入前去重（强制）
- 先检索同主题记录，命中则更新，不重复新建。
- 同一结论禁止在多个文件重复拷贝。

---

§5 多会话调度防跑偏（强约束）

- 调度消息必须带最小上下文头：
  [group_name|scope|target|task_id]
- 子会话回调必须带同一 task_id。
- 未带 scope/target 的执行类任务，不得下发。
- 负责人会话在调度前必须做一次上下文回显，确认后再派发。

---

§6 上下文压缩恢复协议（第一条必须执行）

压缩后第一条执行恢复流程：
1) 读取团队级最近 3 条记忆
2) 读取全局级最近 3 条纠错记录
3) 输出 3 行恢复摘要：
- 当前目标
- 当前上下文（group_name/scope/target）
- 下一步动作

恢复摘要完成后才允许继续执行任务。

§8 安全与质量底线

- 禁止猜测 API 路径、参数、命令。调用前先读对应 Skill 手册。
- Skill 能完成的操作必须优先走 Skill，不绕过。
- 不确定时先查证再行动，禁止无事实依据的回答。
- 批量任务一次执行到可验证节点再汇报，避免高频打断。

§8.1 Skill 优先与 Skill 演进（重点）

- Skill 是执行流程的一部分，不是附属文档。凡是 Skill 能完成的动作，优先走 Skill。
- 当发现“执行不流畅、步骤重复、规则不足”时，优先补充或修订对应 Skill，再继续任务。
- 新增或修改 Skill 后，需在记忆库写一条变更摘要：问题、改动点、适用场景、注意事项。
- 全局规则仅定义“流程骨架”，细节约束优先沉淀到 Skill，避免全局规则过载。

§9 记忆库与知识库写作规范（简版）

记忆（memory）
- 记录“可直接复用的经验结论”，短小、可执行。
- 推荐结构：场景/问题/结论/约束/验证时间。

知识（knowledge）
- 记录“稳定事实与流程”。
- 推荐结构：背景/步骤/参数/边界/参考。

命名建议（避免重复）
- <域>-<主题>-<关键词>.md
- 例如：运维-会话恢复-resume循环.md

§10 规则自治

- 发现规则冲突、顺序不清、触发过度，先提优化建议，再按确认更新。
- 规则更新后必须写入一条纠错记忆：改了什么、为什么改、影响范围。
